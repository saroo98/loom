Sovereign by architecture

Nothing leaves your machine. Audit it yourself.

Straight answers

FAQ

Does anything get sent anywhere?

No. Run the audit above — zero network-capable imports, and the only subprocess targets are git and the Python test runner. The only network activity in the system is git, against remotes you configure. This page makes zero third-party requests too.

Do my lessons improve the upstream repo?

No. Your retro feeds your Loom — that's the point. Every install is owner-operated and instances diverge on purpose.

Is this a framework I have to adopt?

It's files: markdown guidance plus small standard-library Python tools. Small tasks skip the ceremony entirely; the artifact matrix exists so you only produce what has a consumer.

How do I know an agent actually follows it?

The mechanical fraction is enforced by a 25+ check linter (gates refuse to open on errors; a pre-commit guard blocks defective packs). The judgment fraction is enforced by gates that demand a review file with cited scores. The honesty fraction is enforced by outcome ledgers — predictions in writing, checked against reality.

Can it call itself production-ready?

Not by itself — literally. Loom ships a falsifiable 1.0 scorecard and a standing rule that no Loom may score itself: only a non-author session, against real project evidence. A fresh install starts unproven on purpose.

Can I publish my own Loom?

Yes — the same leak-firewalled publisher that produced the repository is in your tools. Allowlist, forbidden-token firewall, link check, suite-in-output.
