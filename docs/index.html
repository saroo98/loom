02 / ONE REQUEST, FULL SYSTEM

The surface stays small.
The reasoning does not.

Every stage is internal. The owner asks once. Loom decides what rigor the work deserves and exposes only consequential uncertainty.

Read the agent kernel
  1. 01

    RESOLVE

    Find the actual target.

    Project identity, configuration, installation, lifecycle, and authority are established before planning begins.

  2. 02

    SURVEY

    Bind the plan to the present.

    Committed, staged, unstaged, untracked, ignored runtime, and time-drifted state become one sealed input.

  3. 03

    ROUTE

    Spend rigor where consequence lives.

    Tier S stays compact. Larger work receives only the plans, work orders, and gates its consumers need.

  4. 04

    DISCOVER

    Use the domain's invariants.

    Loom never substitutes web conventions for accounting, 3D, firmware, research, data, mobile, or an unknown field.

  5. 05

    SEAL

    Make planning precede building.

    The exact artifact set, work-order topology, G1 review, touches, and acceptance evidence become tamper-evident.

  6. 06

    VERIFY

    Close the loop in reality.

    Real-medium evidence, drift detection, bounded retries, and compact receipts replace “looks good” completion.
