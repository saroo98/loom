# Simple adaptive experience

Loom keeps one visible surface: `/loom` plus ordinary language. Routine success returns the sealed
`owner_message.human` value in no more than two lines. That message always states consequence,
verification and freshness, reversibility, one next action, and a short receipt ID. A stopped or
uncertain run presents exactly one decision and one recommendation. Full machine receipts remain
available through ordinary inspect or explain requests.

## Automatic continuation

Automatic continuation is a closed mechanical decision. It is permitted only when work is
reversible, non-destructive, within current scope, ordinary in consequence, free of external effects
and cost, currently evidenced, and verifiable before harm. Deletion, credentials, production or
external effects, spending, privacy expansion, privilege, and legal or safety judgment require the
owner's explicit request. The authority receipt is content-bound inside the action. It describes
authority but cannot replace lifecycle or freshness gates.

## Proportional effort

Effort starts from observations rather than a label: atomic outcomes, touched scope, new boundaries,
consequence, reversibility, domain coverage, dependency reach, novelty, and repository health. These
observations produce explicit obligations. S, M, L, and XL remain compatibility labels only. A small
task is promoted when any small-task premise fails, including unknown domain coverage, more than one
outcome, more than five touches, new boundaries, irreversibility, material consequence, or unknown
repository health.

Static guidance is compiled into bounded, content-addressed planning atoms. Inactive modules emit no
bytes into the plan contract. Cache classes are versioned for static guidance, host adapters, project
routing, domain authority, owner selection, and provider prefixes. Each class is bound to exact
generations and invalidates only its dependency subtree. Cache state never authorizes work.

## Learning that cleans itself up

Selected memory must identify the decision or verification obligation it affected. Selection alone
does not earn credit. Verified help, harm, neutrality, ambiguity, and rejection remain separate.
Serious verified harm quarantines a record immediately.

Active context, retained records, cold archive bytes, effects, events, devices, quarantine, backups,
and receipts are independently bounded. Dormant learning contributes zero active context. Legacy
`stale` memory materializes as `revalidation-required`. Forgetting remains terminal and deletion
floors dominate merge and restore.

`/loom show loom health` exposes counts and bounds, never memory bodies: selected and dormant
learning, recent benefit and harm, preference conflicts, forgetting commitments, backup and device
state, quarantine, and evidence sufficiency. Natural-language inspect, explain, correct, retain, and
forget requests remain on the same surface.

## Evidence boundary

The local state machines, migrations, bounds, and regression simulations are mechanically tested.
Universal speed, causal long-term owner benefit, real-host auto-activation quality, and unfamiliar-user
comprehension remain **[UNVERIFIED]** until their specified external experiments produce current,
subject-bound evidence.
