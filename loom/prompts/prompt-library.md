# Internal prompt contracts

These are internal agent contracts behind the single `/loom <request>` surface. They are not
additional user commands.

## Plan

Use the resolved target, tier, domain adapters, bounded owner context, and artifact matrix. Preserve
the request verbatim. Produce only artifacts with a named consumer/decision. Seal G1 before any
implementation and return measured usage separately from semantic evidence.

## Execute one work order

Run the pre-work freshness check. Claim exactly one ready work order. Respect declared touches and
hard stops. Complete every acceptance criterion using the real medium, update close-out evidence,
and let the lifecycle gate seal changed paths. A no-op or out-of-scope change is a blocked result.

## Repair/resume

Establish the current world state first. Determine affected plan sections from dependency evidence,
revalidate them, and checkpoint the repaired state. Do not execute during the repair pass.

## Review/close

Review evidence independently of the author when the gate requires it. Close only when every work
order has a sealed completion, required release checks pass, rollback is credible, and remaining
unknowns are explicitly accepted by the proper authority.
